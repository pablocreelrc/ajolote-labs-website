# Adversarial UX/UI craft review → path to top-notch. v6 · 2026-07-15

Two adversarial craft critics reviewed the current build (post gradient/green/stagestrip fixes) against a best-in-class bar (Linear / Vercel / Stripe / Awwwards-tier). Critic A = interaction & motion director (scored 68/100). Critic B = visual systems & typography director (scored 70/100). Scope: **UX/UI craft only** — positioning/copy is a separate later round. Evidence: 14 screenshots in this folder (D0-D6 desktop, M1-M7 mobile). Contested claims were verified by me against the live build; verdicts marked below.

## Headline verdict

Both critics land in the same place independently: **a well-executed, coherent dark-cinematic site that reads as "carefully art-directed," not "systematized / site-of-the-day."** The two things holding it back from top-notch are (1) the signature **platform diagram is the weakest craft on the page**, and (2) the **3D brain — the literal product metaphor — is the deadest layer on the site.** Fixing those two, plus systematizing type and varying the motion, is the path.

---

## Confirmed high-value findings (both critics, verified)

### T1. The platform diagram is under-designed (the signature section is the weakest). CONFIRMED
- Desktop (D3): the four node labels don't share an axis (Context/AI-Employees left-aligned, Models/Governance right-aligned) so the eye ping-pongs; the brain→node connector lines are **verified at 0.3 opacity in a dim border color** so they barely read as connections; and a 3-paragraph text column competes with the diagram in the same 100svh panel (two information structures crammed together).
- Mobile (M4): the desktop's best moment (signal riding traces into the brain) degrades to a static vertical card list joined by ▼ chevrons.
- REFUTED sub-claim: "the brain card floats off-center" — it is centered at the grid center (x=1015), symmetric. Not a defect.
- Elevation: commit to one geometry (radial hub-and-spoke with visible glowing directional connectors, OR a clean 2×2 with the brain as a labeled center seam), align all node labels to a shared system, promote the connectors to first-class animated elements, and move the left text column to a stacked intro *above* the diagram so it gets full width.

### T2. The 3D brain is inert wallpaper. CONFIRMED (architecture)
Verified: the background brain has no cursor/scroll reactivity (no input handlers; only opacity ramps per section). For a firm whose entire pitch is "we build the brain," the brain is the least alive element on the page.
- Elevation: give it agency — cursor parallax/repulsion on desktop, particle energy tied to scroll velocity, and (highest-value) a visible **pulse fired in sync with the platform diagram's console beats** so it looks like it's *thinking* during the reveal. Port that firing to mobile.

### T3. Mobile particle field bleeds through the platform cards. CONFIRMED (visible M4)
The static poster renders at card-edge sharpness in the gutters between the stacked platform cards — reads as a pixelated rendering artifact, the single ugliest frame in the set. (The earlier `.plat2` scrim only covers the text blocks, not the card stack.)
- Elevation: drop the brain to near-zero opacity behind the stacked mobile cards, or add a blurred/solid scrim behind the card stack.

### T4. Motion is one-note. CONFIRMED (A4)
Every section = word-cascade headline + fade-translateY(16px) body. By section 3 it's fully predictable and stops registering. No escalation, no signature beat besides the diagram.
- Elevation: give each section a distinct motion tied to its meaning (the "scattered knowledge" thesis should have text/particles *converging* on scroll-in; cases should count-up or line-draw on entry). Retire the uniform fade-up default.

### T5. Type scale + micro-label grammar are ad hoc, not a system. CONFIRMED (B3, B4)
- Headline sizes (~hero 120 / CTA 90 / thesis 85 / platform-cases 64) have no visible modular ratio — 4-5 arbitrary display sizes.
- Eyebrow grammar is three systems: dot/equals ("NEXT.DEPLOY = YOU"), snake_case ("WHY_WE_EMBED"), plain caps ("ONE BRAIN OVER EVERYTHING YOU RUN"), and Cases has none.
- Elevation: collapse to a 3-tier modular display scale (hero/CTA max, section-head mid, sub min) and one label grammar on every section. NOTE: the label-grammar text change brushes the deferred copy round — flag for Pablo; the type-scale part is pure craft.

### T6. Snap-panel voids + no shared vertical grid. CONFIRMED (B6, B7, A11)
Content is top-anchored in 100svh snap panels, leaving dead lower thirds (D2, D4); sections also enter at inconsistent Y positions (eyebrows start at ~140/263/170), so scrolling between panels feels jittery, not locked.
- Elevation: one vertical anchoring rule (optical-center or fixed top grid line) + a shared eyebrow→H1→body spacing token reused across sections.

## Verified single-critic findings worth doing

### T7. Preloader spends the most expensive 3.5s on decoration. (A1)
The boot converges the brain, then *recedes it to a faint background* — you pay 3.5s to build a thing then hide it. "Tap to skip" admits it's in the way.
- Elevation: cut to ≤1.2s and make it a **shared-element handoff** — the converged brain settles directly into its hero position/opacity, so the boot *is* the hero's entrance, not a gate before the value prop. (Note: it was already reduced 7s→3.5s previously; this is the next step.)

### T8. Case card border-beam creates a false "selected" state. DOWNGRADED (B8/A5)
Both cards run an always-on rotating conic beam; at any instant they're at different phases, so one looks brighter/"selected." It's animation phase, not a static asymmetry — but the always-on beam is the cause.
- Elevation: make the beam **hover-reactive** (idle-dim, brighten + sweep to cursor on hover) and add a real card hover state (lift + "read case →"). Reserve the always-on beam for one primary card at most.

### T9. Nav is a static slab with no scroll-state. (A10)
Same bar over the near-black hero and over the brighter platform/closing sections; no condense, no backdrop-blur kick-in, and the "● ONLINE" pill doesn't pulse.
- Elevation: condense the header on first scroll (shrink padding, add blur + hairline), pulse the pill.

### T10. No section wayfinding despite a paginated model. (A9)
A discrete snapped section-per-panel structure with no progress indicator/section dots — the model that most demands a spine has none, which makes snap feel like a trap.
- Elevation: a minimal right-edge section-dot rail or a thin top progress bar across the 5 panels.

### T11. Mobile nav drawer + poster are budget-tier. (A13, A8)
Drawer uses a max-height slide (non-linear, links appear all at once); the mobile poster is a hard grainy frozen frer asset under animated foreground.
- Elevation: transform-based drawer slide with staggered link items + burger→X morph; higher-fidelity poster with a slow breathe so it isn't visibly dead.

### T12. Smaller craft items.
- Body measure narrow (~340px) under wide headlines → ragged left-column silhouette; widen toward ~60-66ch (B12).
- Console: add a blinking cursor on the last line + a scale-in pop on each ✓ (console already streams live — verified — so this is polish, not a fix) (A14).
- `/cases` route: tall void between back-link and headline; tighten to section rhythm (B14).
- Primary CTA (used 6×) deserves a signature micro-interaction (fill-sweep / magnetic pull), not just brighten (A12).
- Mobile footer link indentation looks inconsistent — quick check/align during implementation (B11).

At the natural reading position the headline sits at 93px, clearing the 58px nav; it's only clipped at the absolute page-bottom (when you're looking at the footer). Cheap `scroll-margin`/bottom-padding tidy-up, not a readability bug.
- **B1 "brain card floats off-center"** → REFUTED (centered/symmetric).
- **A14 "console may hard-appear, illusion collapses"** → REFUTED. Verified it streams one line per beat live.

## Path to top-notch — ranked roadmap

**Tier 1 — quick craft wins (CSS-mostly, low risk, high polish-per-effort):**
1. T3 mobile particle bleed behind platform cards (scrim / near-zero opacity).
2. T9 nav scroll-condense + pulse the ONLINE pill.
3. T8 case border-beam → hover-reactive + real card hover state.
4. T6 shared vertical grid + anchoring on snap panels (eyebrow→H1→body token).
5. T5b type scale → 3-tier modular (pure craft half of T5).
6. T12 batch: body measure, /cases void, footer indent, console cursor/✓ pop, B5 bottom-padding tidy.

**Tier 2 — signature bets (bigger, this is what buys "top-notch"):**
7. T1 rebuild the platform diagram (one geometry, aligned labels, real animated connectors, full-width) + mobile scroll-driven version.
8. T2 make the brain react (cursor + scroll-velocity + pulse on diagram beats) — the single highest-ambition move; directly on-message ("we build the brain").
9. T7 preloader → shared-element hero handoff, ≤1.2s.
10. T4 per-section motion signatures + T10 section-progress spine.

**Decisions for Pablo (change direction, not just polish):**
- **Scroll model** (A3): keep desktop mandatory snap, or relax to `proximity` / drop it for scroll-driven choreography? Both critics see mandatory snap as the biggest usability-vs-motion tension.
- **Preloader philosophy** (T7): is a boot animation worth any first-impression cost, or should the value prop paint instantly?
- **Label grammar** (T5): unifying the eyebrow grammar changes label wording → touches the deferred copy round; do it now as a system decision or defer.
