# v5 Redesign — Information Architecture (sections + information)

**Status:** DRAFT for Pablo to react to. Derived from `vision/` + the goals in `00-goals.md`. This defines *what each section says and why* — before any visual direction. Variants get proposed only after this is locked.

For each section: **Purpose** · **The buyer question it answers** · **Key information** · **Conversion role** · **Motion opportunity** (where anime.js might later earn its weight).

---

## 0. Nav (persistent)
- **Purpose:** wayfinding + always-available CTA.
- **Answers:** "How do I get in touch / what's here?"
- **Info:** brand mark, section anchors (Services, Cases, Contact), Book-a-call CTA.
- **Conversion:** persistent Calendly button.
- **Motion:** minimal — sticky blur, active-section indicator.

## 1. Hero — the promise
- **Purpose:** in 5 seconds, say what we do and why it matters.
- **Answers:** "What is this and what do I get?"
- **Info:** category line (infrastructure operator); H1 (money lost to manual ops → outcome); sub (we build it, run it, never leave); a preview of the "after" state (the deliverable card / live system).
- **Conversion:** primary Book-a-call.
- **Motion:** the signature moment candidate — system "boot-up" / terminal coming online.

## 2. The problem / why now
- **Purpose:** name the pain and the timing so the offer has a reason to exist.
- **Answers:** "Why should I care, and why now?"
- **Info:** manual operations bleed money; every company will run AI agents; agents are useless without the connective infrastructure — someone has to build and run it.
- **Conversion:** none (sets up the offer).
- **Motion:** subtle reveal; possibly a "manual vs. operated" contrast.

## 3. How we work — the 3 layers
- **Purpose:** the operating model, not a feature list.
- **Answers:** "What do you actually build?"
- **Info:** **The Brain** (unify every tool into one source of truth) → **AI Employees** (agents that run the work 24/7) → **Power Users** (your team commands them while we keep operating underneath).
- **Conversion:** soft (see-how-we-work → engagement section).
- **Motion:** layered stagger; each layer "wires" to the one above.

## 4. The engagement shape — 5 stages  ← (currently MISSING from the live site)
- **Purpose:** show this is a durable operating relationship, not a project. This is the operator proof.
- **Answers:** "What happens if I sign, and do you leave?"
- **Info:** Discovery → Node setup → Agents → Outward MCP → **Maintain (forever)**. Emphasize stage 5 = the business model.
- **Conversion:** "this is what a call kicks off."
- **Motion:** horizontal/scroll-driven progression through stages; stage 5 never "completes."

## 5. Proof — case studies
- **Purpose:** evidence the outcomes are real.
- **Answers:** "Has this worked for anyone like me?"
- **Info:** anonymized, outcome-led cards (cross-border spirits; enterprise loyalty). Lead with the business result, not vanity counts.
- **Conversion:** medium — proof primes the call.
- **Motion:** number count-ups; reveal on scroll.

## 6. Who this is for
- **Purpose:** self-qualification — the buyer recognizes themselves (or rules themselves out).
- **Answers:** "Is this for a company like mine?"
- **Info:** verticals (CPG/spirits, healthcare/clinics, loyalty/B2B2C, industrial SME); company shape (50–500, ops-heavy, multi-tool stack).
- **Conversion:** medium.
- **Motion:** light.

## 7. Differentiation — operator vs. the alternatives
- **Purpose:** kill the "isn't this just a consultancy / SaaS?" objection.
- **Answers:** "How is this different from an agency or a tool?"
- **Info:** translated version of the thesis table — vs. SaaS, vs. consultancy, vs. system integrator. The "you own it, we operate it, we never leave" line.
- **Conversion:** medium — removes the last doubt before the CTA.
- **Motion:** comparison reveal.

## 8. Closing CTA + footer
- **Purpose:** final ask + housekeeping.
- **Answers:** "How do I start?"
- **Info:** one strong Book-a-call; minimal footer (contact, anonymized vibe, status line). No founder/team/geo.
- **Conversion:** primary Book-a-call.
- **Motion:** minimal.
